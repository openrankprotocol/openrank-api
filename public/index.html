OpenRank API

REST API for OpenRank data across Discord, GitHub, Telegram, and X

Endpoints

Discord

Available files: ritual

GET/discord/{file_name}
GET/discord/{file_name}/seed
GET/discord/{file_name}/server_id
GET/discord/{file_name}/scores

GitHub

Available files: bitcoin

GET/github/{file_name}
GET/github/{file_name}/seed
GET/github/{file_name}/ecosystem
GET/github/{file_name}/scores

Telegram

Available files: decentraliseddotco

GET/telegram/{file_name}
GET/telegram/{file_name}/seed
GET/telegram/{file_name}/channel_id
GET/telegram/{file_name}/scores

X

Available files: ritual-community

GET/x/{file_name}
GET/x/{file_name}/seed
GET/x/{file_name}/community_id
GET/x/{file_name}/scores

Query Parameters

For /scores endpoints, use these optional query parameters:

  • start - Starting index for pagination (default: 0)
  • size - Number of items to return (default: all)

Examples

Get complete Discord dataset:

curl https://api.openrank.com/discord/ritual

Response:

{
  "category": "socialrank",
  "server_id": "1210468736205852672",
  "seed": [
    { "i": "josh.simenhoff", "v": 0.25 },
    { "i": "0xqt3.14", "v": 0.25 },
    ...
  ],
  "scores": [
    { "i": "josh.simenhoff", "v": 1000.0 },
    { "i": "0xqt3.14", "v": 986.31 },
    ...
  ]
}

Get Discord server ID:

curl https://api.openrank.com/discord/ritual/server_id

Response:

{
  "server_id": "1210468736205852672"
}

Get GitHub seed data:

curl https://api.openrank.com/github/bitcoin/seed

Response:

{
  "seed": [
    { "i": "ethereum/go-ethereum", "v": 0.05 },
    { "i": "ethereum-optimism/optimism", "v": 0.05 },
    { "i": "op-rs/kona", "v": 0.05 },
    ...
  ]
}

Get GitHub ecosystem:

curl https://api.openrank.com/github/bitcoin/ecosystem

Response:

{
  "ecosystem": "Crypto"
}

curl "https://api.openrank.com/discord/ritual/scores?start=0&size=10"

curl "https://api.openrank.com/x/ritual-community/scores?start=20&size=5"

Response:

{
  "scores": [
    { "i": "user_20", "v": 542.12 },
    { "i": "user_21", "v": 538.45 },
    { "i": "user_22", "v": 535.67 },

Response Format

All responses are returned in JSON format. Score entries follow this structure:

{
  "i": "identifier",
  "v": 123.45
}
